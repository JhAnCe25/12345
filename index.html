Classification d'Activités Humaines avec Optimisation Convexe

Reconnaissance d'activités physiques à partir de capteurs de smartphone avec Condat–Vũ et ADMM linéarisé

Introduction Algorithmes Dimensions

Introduction

Objectif du projet : Développer un classificateur multiclasse pour la reconnaissance d'activités humaines à partir des données de capteurs de smartphones (base de données UCI HAR).

Contexte

Les données proviennent de 30 volontaires effectuant 6 activités différentes avec un smartphone Samsung Galaxy S II attaché à la taille :

MARCHER

Walking

MARCHER EN MONTÉE

Walking Upstairs

MARCHER EN DESCENTE

Walking Downstairs

ASSIS

Sitting

DEBOUT

Standing

COUCHÉ

Laying

Prétraitement des Données

Pour gérer la haute dimensionnalité des données (561 features originales), nous utilisons :

Réduction de dimension avec PCA :

561 features originales → 128 features sélectionnées

XPCA = PCA(Xoriginal, n_components=128)

Cette réduction permet :

  • De réduire le bruit et la corrélation entre features
  • D'améliorer l'efficacité computationnelle
  • De préserver 95%+ de la variance des données
  • D'éviter le sur-apprentissage

Problème d'Optimisation

Nous résolvons un problème de classification multinomiale avec régularisation L1 sur les poids des features aléatoires, mais pas sur le biais :

minW,b f(W,b) + λ‖W‖1

où f(W,b) = (1/n) ∑i=1n l(softmax(s(xi; W,b)), yi)

s(x; W,b) = φRFF(x)TW + b ∈ ℝC

softmax(s)c = exp(sc) / ∑j=1C exp(sj)

l(p, y) = -∑c=1C yc log(pc)

Approximation de noyau RBF via Random Fourier Features

Pour éviter le coût du noyau RBF, nous utilisons une approximation linéaire :

φRFF(x) = √(2/M) cos(WRFFx + bRFF) ∈ ℝM

où WRFF ∈ ℝM×d ∼ N(0, σ²), bRFF ∈ ℝM ∼ U[0, 2π]

Note : Pour inclure un biais dans le classifieur, on ajoute une colonne de 1 à la matrice de features :

Φ = [φRFF(X), 1] ∈ ℝn × (M+1)

Θ = [W; b] ∈ ℝ(M+1) × C (paramètres à apprendre)

Seul W (les M premières lignes) est régularisé par L1, b (dernière ligne) ne l'est pas.

Dimensions des Variables

Variable Dimension Description
xi ℝd Donnée d'entrée originale (d features)
φRFF(xi) ℝM Features aléatoires (M features RFF)
Φ = [φRFF(X), 1] ℝn×(M+1) Matrice des features avec colonne de biais
W (poids RFF) ℝM×C Poids des features, régularisés par L1
b (bias) ℝC Biais par classe, non régularisé
Θ = [W; b] ℝ(M+1)×C Paramètres complets à apprendre
yi ℝC Label encodé one-hot
s(xi; W,b) ℝC Scores = φRFF(xi)ᵀW + b
WRFF ℝM×d Poids RFF (fixés aléatoirement)
bRFF ℝM Biais RFF (fixés aléatoirement)

Algorithmes d'Optimisation

Calcul de la constante de Lipschitz L

Pour déterminer les pas d'apprentissage, nous avons besoin de la constante de Lipschitz du gradient de f :

L = 0.25 × ‖Φ‖² / n

La borne 0.25 vient du fait que pour la fonction logistique/softmax, la dérivée seconde est bornée par 1/4.

‖Φ‖ est la norme spectrale (plus grande valeur singulière) de la matrice Φ, estimée par itération de puissance (20 itérations).

Algorithme Condat–Vũ

Algorithme primal-dual pour problèmes composites f + g

Problème : minΘ f(Θ) + g(Θ) avec Θ = [W; b]

f différentiable (perte softmax), g(Θ) = λ‖W‖₁ (non-différentiable, n'affecte pas b)

Étapes itératives :

1. yk+1 = proxτg(Θk - τ∇f(Θk))

2. Θk+1 = Θk + ρ(yk+1 - Θk)

Conditions : τ ∈ (0, 2/L), ρ ∈ (0, 2)

Proximal pour g : Applique le seuillage doux seulement aux lignes de W (pas à b).

Paramètres pratiques

τ : τ = 0.9/L (proche de 1/L avec marge de sécurité)

ρ : ρ = 1.5 (surrelaxation modérée)

Pourquoi 0 < ρ < 2 ?

Condition de convergence des méthodes de relaxation. Permet d'accélérer (ρ>1) ou stabiliser (ρ<1) sans compromettre la convergence.

Algorithme ADMM Linéarisé

Alternating Direction Method of Multipliers avec linéarisation de f

Problème reformulé : minΘ,Z f(Θ) + g(Z) s.c. Θ = Z

avec Θ = [W; b], Z = [Z_W; Z_b] et g(Z) = λ‖Z_W‖₁ (Z_b libre)

Linéarisation de f autour de Θk :

f(Θ) ≈ f(Θk) + ⟨∇f(Θk), Θ-Θk⟩ + (1/(2η))‖Θ-Θk‖²

Étapes itératives :

α = 1/(1/η + ρ)

Θk+1 = α[(1/η)Θk - ∇f(Θk) + ρ(Zk - Uk)]

Zk+1 = proxg,1/ρ(Θk+1 + Uk) (seuillage sur Z_W seulement)

Uk+1 = Uk + (Θk+1 - Zk+1)

Critères d'Arrêt

Résidu primal : ‖Θ - Z‖F < εp

Résidu dual : ρ‖Zk+1 - Zk‖F < εd

Paramètres pratiques

η : η = 0.9/L (proche de 1/L avec marge de sécurité)

ρ : ρ = 1.0 (valeur typique pour ADMM)
